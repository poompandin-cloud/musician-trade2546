# อัปเดตให้เจ้าของงานเห็นงานของตัวเองในหน้าประกาศ

## สิ่งที่ได้แก้ไข:

### 1. ✅ อัปเดต App.tsx - แก้ไข Query Logic
**ก่อนหน้านี้**: แสดงเฉพาะงานที่ `status = 'open'` หรือ `null`
```sql
.or('status.eq.open,status.is.null')
```

**ปัจจุบัน**: แสดง 1) งานที่เปิดทั้งหมด 2) งานของตัวเองทุกสถานะ
```sql
.or('status.eq.open,status.is.null,user_id.eq.' + (session?.user?.id || ''))
```

### 2. ✅ อัปเดต NearbyGigs.tsx - เพิ่ม UI สำหรับเจ้าของงาน
- **Visual Indicator**: กรอบสีส้ม + พื้นหลังอ่อนสำหรับงานของตัวเอง
- **Badge "ประกาศของคุณ"**: แสดงว่านี่คืองานของเรา
- **Toggle Button Text**: เปลี่ยนข้อความปุ่มเป็น "ดู/ซ่อนรายชื่อผู้สมัคร"

### 3. ✅ ปรับปรุงประสบการณ์ผู้ใช้
- **Mobile Responsive**: ทุก UI รองรับมือถือ
- **Clear Visual Hierarchy**: แยกความแตกต่างระหว่างงานของตัวเองกับคนอื่น
- **Easy Testing**: เจ้าของงานสามารถทดสอบระบบได้ทันที

## การทำงานของระบบ:

### สำหรับเจ้าของงาน:
1. **เห็นงานของตัวเอง** → แม้งานจะถูกปิด (status = 'closed')
2. **ระบุงานของตัวเอง** → กรอบสีส้ม + badge "ประกาศของคุณ"
3. **จัดการผู้สมัคร** → กด "ดูรายชื่อผู้สมัคร" ได้ทันที
4. **ทดสอบระบบ** → สามารถทดสอบการยืนยัน/ปฏิเสธได้

### สำหรับนักดนตรีคนอื่น:
1. **เห็นเฉพาะงานที่เปิด** → ไม่เห็นงานที่ถูกปิด
2. **สมัครงานได้** → กด "รับงานนี้" ตามปกติ
3. **ไม่รบกวน** → ไม่เห็นงานที่ปิดแล้ว

## วิธีทดสอบ:

### 1. สร้างงานทดสอบ:
1. ล็อกอินด้วยบัญชีใดๆ
2. ไปที่หน้า "ประกาศหาคนแทน"
3. สร้างประกาศงานใหม่

### 2. ทดสอบเห็นงานของตัวเอง:
1. ไปที่หน้า "หาคู่ประกาศงาน"
2. ควรเห็นงานที่สร้างไว้ (กรอบสีส้ม)
3. กด "ดูรายชื่อผู้สมัคร"

### 3. ทดสอบระบบสมัครงาน:
1. ล็อกอินด้วยบัญชีอื่น
2. กด "รับงานนี้" ในงานของคุณ
3. กลับไปทดสอบการยืนยันผู้สมัคร

### 4. ทดสอบหลังปิดงาน:
1. ยืนยันผู้สมัคร 1 คน
2. งานจะถูกปิด (status = 'closed')
3. คุณยังคงเห็นงานนี้ (แต่คนอื่นไม่เห็น)
4. สามารถจัดการผู้สมัครต่อได้

## ประโยชน์ของการอัปเดตนี้:
- ✅ **ทดสอบได้ง่าย**: เจ้าของงานสามารถทดสอบระบบได้ทันที
- ✅ **จัดการงานได้**: สามารถดูและจัดการงานของตัวเองตลอดเวลา
- ✅ **ไม่รบกวนผู้อื่น**: คนอื่นไม่เห็นงานที่ปิดแล้ว
- ✅ **UI ชัดเจน**: แยกความแตกต่างระหว่างงานของตัวเองกับคนอื่น
